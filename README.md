# Telegram Userbot — Heroku Deploy

A minimal **Telethon userbot** that logs into your Telegram account and
updates your profile name to **pairing ⚡** (with a custom emoji) every
time it starts.

---

## Files

| File | Purpose |
|---|---|
| `userbot.py` | Main bot — entry point |
| `generate_session.py` | Run **locally once** to get your `STRING_SESSION` |
| `requirements.txt` | Python dependencies |
| `Procfile` | Tells Heroku to run a worker dyno |
| `runtime.txt` | Pins Python 3.12.3 on Heroku |
| `app.json` | Heroku app metadata & config vars |
| `.gitignore` | Keeps secrets & session files out of git |

---

## Step 1 — Generate your STRING_SESSION (local, one-time)

> You need Python and `telethon` installed locally.

```bash
pip install telethon
python generate_session.py
```

Enter your phone number and the Telegram login code when prompted.  
Copy the long string that is printed — that is your `STRING_SESSION`.

---

## Step 2 — Push to GitHub

```bash
git init
git add .
git commit -m "Initial userbot"
git remote add origin https://github.com/your-username/your-repo.git
git push -u origin main
```

---

## Step 3 — Deploy to Heroku

```bash
# Install Heroku CLI if you haven't already
heroku login
heroku create your-app-name

# Set config vars
heroku config:set API_ID=35057620
heroku config:set API_HASH=010824c75f6d2116f660933cf4536d20
heroku config:set STRING_SESSION=your_string_session_here

# Deploy
git push heroku main

# Scale the worker dyno (web dynos won't work for a userbot)
heroku ps:scale worker=1

# Watch logs
heroku logs --tail
```

---

## Config Vars

| Variable | Description |
|---|---|
| `API_ID` | From [my.telegram.org](https://my.telegram.org) |
| `API_HASH` | From [my.telegram.org](https://my.telegram.org) |
| `STRING_SESSION` | Generated by `generate_session.py` |

> ⚠️ **Never commit `STRING_SESSION` or `.session` files to git.**  
> The `.gitignore` already blocks `.session` files, but double-check before pushing.

---

## Extending the bot

Add new event handlers inside `userbot.py` under the `# Event handlers` section:

```python
@client.on(events.NewMessage(pattern="/hello"))
async def hello(event):
    await event.reply("Hi!")
```
